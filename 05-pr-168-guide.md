## Description
Fixed incorrect jarfile name in the RESTful Web Service guide that was causing "Error: Unable to access jarfile" errors.

## Problem
The guide instructed users to run:
```bash
java -jar target/gs-rest-service-0.1.0.jar
```

But Maven actually generates:
```bash
target/rest-service-complete-0.0.1-SNAPSHOT.jar
```

## Solution
Updated the documentation to use the correct filename generated by Maven:
- Changed: `gs-rest-service-0.1.0.jar` → `rest-service-complete-0.0.1-SNAPSHOT.jar`

## Testing
- [x] Verified the correct jarfile name from `complete/pom.xml`
